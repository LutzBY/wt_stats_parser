# wt_stats_parser
War Thunder battle results parser
TLDR - Принимает по ctrl+c стандартизированный отчет о миссии из WT (генерится в меню сообщений игры), парсит его и сохраняет в xlsx / выводит в окошко tkinter. 
Дополнительно получает/формирует отсутствующую в отчете информацию (страна, тип миссии, БР) на основании БД техники, полученной с wiki.warthunder.com

Версия 3:
1. parse_battle_stats() - Получает ч/з pyperclip отчет и парсит его с использованием re
2. save_to_excel() - Получает от 1. словарь и пишет/обновляет xlsx файл
3. BattleAnalyzer - класс функций. Загрузка и подготовка/нормализация/очистка json БД, получение страны и БР, аппроксимация типа боя исходя их строки vehicles (используемая техника). Получения флага страны, для отображения в tkinter
4. WTapp - блок tkinter. Кнопка, текстовые поля, вывод print()
5. TextRedirector - stdout, перехват и передача терминала в окно tkinter
6. start_global_listener() - листенер для работы перехвата ctrl+c. Проверяет запущен ли WT и следует ли запускать нажатие кнопки

Из костылей на данный момент:
- getpass используется для подстановки "конфигурации" (локаций файлов и положения окна tkinter), позднее нужно реализовать через config
- Ненадежное определение активности окна в windowed конфигурации
- Доработка регулярки (фигурирует ПА, который может быть не у всех пользователей)
- Дополнительная аналитка - подсчет sl/rp/frp без учета бустеров. Вывод среднего по по БР+страна+режим в tkinter
- Лишний print() в окне - заменить на строки tk.title, убрать перехват терминала
- Не отличается ТРБ и ТСБ, не нашел возможности разделить

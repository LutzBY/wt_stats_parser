# wt_stats_parser
War Thunder battle results parser
TLDR - Принимает по ctrl+c стандартизированный отчет о миссии из WT (генерится в меню сообщений игры), парсит его и сохраняет в xlsx / выводит в окошко tkinter. 
Дополнительно получает/формирует отсутствующую в отчете информацию (страна, тип миссии, БР) на основании БД техники, полученной с wiki.warthunder.com
В отчете выводит различную статистику по аналогичным боям. Показывает флажок страны, присваивает значок эффективности в списке техники из боя

Версия 7:
Теперь все это добро работает на трех фреймах tkinter через главное MainApp
0. save_raw_report - сохраняет целиком переданный репорт в txt
1. parse_battle_stats() - Получает ч/з pyperclip отчет и парсит его с использованием re
2. save_to_excel() - Получает от 1. словарь и пишет/обновляет xlsx файл. Дополнительно сохраняет бои за время работы в df_for_session
3. BattleAnalyzer - Класс аналитических функций и методов для работы с БД техники.
Загрузка и подготовка/нормализация/очистка json БД, получение страны и БР, аппроксимация типа боя исходя их строки vehicles (используемая техника). Получения флага страны, для отображения в tkinter. Расчет средних по условиям нация / бр с поправкой на бустеры.
3.11 Функция-аналог parse_battle_stats, но для каждой техники в бою отдельно. Полностью независимо обрабатывает лог и записывает на отдельный лист (vehicles) в xlsx.Формирует также battle_data_vehicles для дальнейшего использования другими статистическими функциями
3.12 Аналитическая функция для StartPageFrame. Открывает xlsx, считает показатели, отдает dict
3.13 Аналитическая функция для SessionSummary. Принимает датафрейм с данными по сессии (df_for_session) и возвращает словарь session_data
4.1 WTapp - Основной фрейм для MainApp. Кнопка запуска, дублирующая ctrl+c. Таблица результатов, содержит конкретный бой, средние по такому же типу и бр, средние по такой же нации и типу и средние в целом по такому же типу боя с поправкой на бустеры.
Выводит название последней записанной миссии, информацию о БР, нации (и ее флаг), использованную технику.
Генерит заголовок вида флаг+результат+миссия, тип боя, бр, использованные бустеры. 
Подсвечивает премиум-технику (золотой текст) и сквадную (зеленый). Также присваивает имени техники значок, соответствующий ее эффективности в бою.
4.2 SessionSummaryWindow. Фрейм для отображения итогов игровой сессии.
4.3 StartPageFrame. Фрейм для отображения статистики по xlsx при запуске программы.
5. MainApp - Управляет главным окном Tkinter и логикой переключения между видами. Регулирует размер и параметры окна и правила смены фреймов.
Также хранит и запускает листенер, который отслеживает активное окно и перехватывает/отправляет ctrl+c

5. TextRedirector - stdout, перехват и передача терминала в окно tkinter
6. start_global_listener() - листенер для работы перехвата ctrl+c. Проверяет запущен ли WT и следует ли запускать нажатие кнопки

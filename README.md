# wt_stats_parser
War Thunder battle results parser
TLDR - Принимает по ctrl+c стандартизированный отчет о миссии из WT (генерится в меню сообщений игры), парсит его и сохраняет в xlsx / выводит в окошко tkinter. 
Дополнительно получает/формирует отсутствующую в отчете информацию (страна, тип миссии, БР) на основании БД техники, полученной с wiki.warthunder.com
В отчете выводит различную статистику по аналогичным боям

Версия 4:
1. parse_battle_stats() - Получает ч/з pyperclip отчет и парсит его с использованием re
2. save_to_excel() - Получает от 1. словарь и пишет/обновляет xlsx файл
3. BattleAnalyzer - класс функций. Загрузка и подготовка/нормализация/очистка json БД, получение страны и БР, аппроксимация типа боя исходя их строки vehicles (используемая техника). Получения флага страны, для отображения в tkinter. Расчет средних по условиям нация / бр и считает средние с поправкой на бустеры
4. WTapp - блок tkinter. Кнопка, текстовые поля, вывод print(). Таблица результатов, содержит конкретный бой, средние по такому же типу и бр, средние по такой же нации и типу и средние в целом по такому же типу боя с поправкой на бустеры
5. TextRedirector - stdout, перехват и передача терминала в окно tkinter
6. start_global_listener() - листенер для работы перехвата ctrl+c. Проверяет запущен ли WT и следует ли запускать нажатие кнопки

Из костылей на данный момент:
- getpass используется для подстановки "конфигурации" (локаций файлов и положения окна tkinter), позднее нужно реализовать через config
- Ненадежное определение активности окна в windowed конфигурации
- Доработка регулярки (фигурирует ПА, который может быть не у всех пользователей). Решить пока не удалось, другие варианты еще менее надежны
- (сделано) Дополнительная аналитка - подсчет sl/rp/frp без учета бустеров. Вывод среднего по по БР+страна+режим в tkinter
- (сделано частично) Лишний print() в окне - заменить на строки tk.title, убрать перехват терминала
- Не отличается ТРБ и ТСБ, не нашел возможности разделить
